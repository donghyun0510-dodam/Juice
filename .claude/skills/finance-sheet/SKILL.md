# 금융 자산 관리 시트 관리 규약

## 탭 구조 (행 1~2가 헤더, 데이터는 월별 1행)

### `저축` — 예적금·연금 월별 잔액
| 컬럼 | 내용 |
|------|------|
| A 월 | `YYYY-MM` |
| B 청년도약계좌 / C 일반적금 / D 주택청약 / E IRP | 각 상품 **월말 잔액** |
| F 합계 | B~E 합 |
| G 입출금(추가입금/출금) | **자동 수식** = `SUM(B:D 이번달) − SUM(B:D 지난달)`. 청년도약·일반적금·주택청약의 월간 잔액 변동을 납입/인출로 자동 인식(IRP 제외). **직접 입력 금지** |
| H 월 손익 = (F 이번달 − F 지난달 − G) | 자동. G가 B~D 변동을 상쇄하므로 **실질적으로 IRP(E) 운용손익만 손익으로 잡힘**(청년/적금/청약은 납입형이라 시장손익 0 가정) |
| I 월 수익률 / J 누적 수익률 | 자동 |

> ⚠️ **저축 탭에서 사람이 입력하는 건 B~E 월말 잔액뿐.** F·G·H·I·J는 전부 수식. 적금 만기·납입 시에도 G를 손대지 말 것(잔액만 바꾸면 G가 자동 반영).

### `주식` — 주식 계좌 평가액
| 컬럼 | 내용 |
|------|------|
| A 월 / B 주식 평가액 | 월말 총평가액(예수금 포함) |
| C 입출금(추가입금/출금) | **`입출금 이력` 탭 합계로 자동 계산** — 직접 입력 금지 |
| D 월 손익 = (B 이번달 − B 지난달 − C) / E 월수익률 / F 누적수익률 | 자동 |

### `입출금 이력` — 주식 계좌 현금 입출금 로그
| A 날짜 | B 금액 | C 메모 |
- 입금 = `+숫자`, 출금 = `−숫자`. 발생 시 **한 줄씩 append**
- `주식` 탭 C열이 이 탭 합계를 월별로 자동 집계
- 예: 월급 입금 `+2,500,000`, 생활비 출금 `−1,500,000`, 적금 납입 출금 `−1,000,000`

### `대출` — 부채 월별 잔액
- 서울시이차보전대출 / 예적금담보대출 각각 잔액·상환원금·이자, 합계·누적 자동

### `기타자산` — 인벤토리 (보증금 등)
- 분류 / 자산명 / 취득일 / 취득가 / 현재가치 / 메모. 전세보증금 등

### `종합` — 월별 순자산 대시보드 (대부분 자동)
| 운용자산(KB+IRP) | 예적금 | 자산합계 | 대출잔액 | 순자산 | 월변동·변동률·누적 |
- `예적금` = 저축 탭 합계, `운용자산` = 주식+IRP, `순자산` = 자산합계 − 대출

### `목표 계산기` — 시나리오 (전부 자동)
- 현재 순자산 기준, 목표 순자산 × 목표 시점 → 필요 연환산 수익률 매트릭스

## 핵심 원칙: 손익 vs 계좌 간 이동 (이게 자주 까먹는 부분)

이 시트의 모든 `월 손익`은 **`(이번달 잔액 − 지난달 잔액 − 입출금)`** 로 계산된다.
따라서 **계좌 사이를 옮긴 돈(이동)은 손익이 아니므로 반드시 입출금 컬럼으로 상쇄**해야 수익률이 왜곡되지 않는다. 순수하게 시장에서 불어난/줄어든 것만 손익이다.

판단 기준:
- **외부에서 자산이 늘거나 줄었나?** (시장 평가손익) → 입출금 0, 잔액만 변동 → 손익으로 잡힘 ✔
- **내 계좌끼리 돈이 이동했나?** (납입·인출·만기) → 양쪽 입출금으로 상쇄 → 손익 0, 순자산 불변 ✔ (단 `저축` 탭 G는 자동 수식이라 **잔액만 바꾸면 입출금이 자동 처리**됨. 사람이 입출금을 직접 적는 건 `입출금 이력`(주식)·`주식` 출금 쪽뿐)

## 주요 작업 레시피

### ① 적금/예금 만기 → 주식 계좌 예수금으로 이동
**두 곳을 처리**한다 (저축 잔액 ↓ / 주식 입금 ↑):

1. **`저축` 탭** (빠져나가는 쪽)
   - 만기된 상품(예: C 일반적금) 잔액을 만기 다음 월에 **0**(또는 잔존분만 남김)
   - → **G(입출금)는 자동 수식**이라 잔액이 0으로 떨어지면 그만큼 자동으로 −인출 처리됨. **사람이 G를 건드리지 않는다.** 손익에도 영향 없음(G가 상쇄).

2. **`입출금 이력` 탭** (들어오는 쪽)
   - 한 줄 append: `날짜 | +만기금액 | "적금 만기"`
   - → `주식` C열에 자동 합산되어, 늘어난 평가액이 수익이 아닌 입금으로 처리

결과: `종합`에서 예적금 ↓ / 운용자산 ↑ 가 상쇄 → **순자산 불변(wash)**, 양쪽 월수익률 정상.
> ⚠️ **둘 다 해야 한다.** 저축 잔액 0만 하고 입출금이력을 빠뜨리면 순자산이 만기금만큼 과소(돈이 증발), 입출금이력만 하고 저축 잔액을 안 비우면 과대(이중계상).
> 💡 **이자 처리**: 입출금이력에 만기금 *전액(원금+이자)*을 +입금하면 이자도 wash라 순자산엔 반영되나 "월 수익"으론 안 잡힌다. 이자를 그달 수익으로 잡고 싶으면 입출금이력에 **원금만** +입금 → 차액(이자)이 주식 탭 그달 수익으로 떨어짐. (저축 탭은 적금 이자를 손익으로 못 잡는 구조 — H가 IRP만 반영)
> 만기금을 주식이 아니라 **다른 적금/소비로** 보내면 ②(입출금 이력) 대신 그쪽 처리만.

### ② 적금 납입 (주식 계좌에서 적금으로)
사용자는 그동안 적금 납입을 **주식 계좌 출금**으로 처리해 옴(입출금 이력에 `−, 메모 "적금"`).
1. `입출금 이력`에 `−납입액, "적금"` append
2. `저축` 탭 해당 상품 잔액 **증액** (G 입출금은 자동으로 `+납입액` 처리 — 직접 입력 X)
→ 만기(①)는 이것의 **정확히 반대 방향**.

### ③ 월말 마감 입력 (매월 말 루틴)
1. `저축`: 각 상품 월말 잔액(B~E)만 입력 (F 합계·G 입출금·H 손익 전부 자동)
2. `주식`: 월말 평가액(B)만 입력 (C는 자동)
3. `입출금 이력`: 그달 입출금이 다 들어갔는지 확인
4. `대출`: 잔액·상환원금·이자 입력
5. `종합`·`목표 계산기`는 자동 — 값만 검증

### ④ 월급/생활비 등 일반 입출금
- `입출금 이력`에 한 줄 append (`+월급`, `−생활비/월세/교육비`)
- 주식 계좌 기준 현금 흐름만 기록. 저축/대출과 무관하면 거기 건드릴 필요 없음
