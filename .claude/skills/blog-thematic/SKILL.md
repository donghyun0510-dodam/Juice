# 도담아빠 블로그 — 테마형 deep-dive 글 작성 규약

일일 리뷰는 `/blog-kr`(국장) 또는 `/blog-us`(미장) 참조. 이 스킬은 비정기 테마 글 포맷.

## 자동 저장 (필수)
초안 출력과 **동시에** 다음 경로에 저장:
- 경로: `C:\Users\dongh\Desktop\주식\blog\draft\`
- 파일명: `YYYY-MM-DD_THEMATIC_<주제>.md` (주제는 kebab-case, 예: `space-etf-comparison`, `spacex-ipo`)

## 6섹션 구조 (가변)

### 1. 도입부 (1~2 단락)
현재 시장 위상 한 줄 + 글의 주제(왜 지금 이 테마?) 한 줄. 마지막 문장은 "~ 알아보겠습니다." 형태로 안내.

### 2. 섹터/종목 지도 (표)
1)/2) 카테고리로 분류한 표. 예: "1) 순수 우주 / 2) 위성 통신". 표 직후 한 줄 요약(전체 시총·대표 종목 비중 등).

### 3. 시나리오·시총 분석 (표 + bullet)
표(시나리오/가치/근거 3컬럼) + 임팩트 bullet 3~4개. 비교 기준(전체 시장 대비 비중·기존 대장주와의 위계 등)을 수치로.

### 4. 분화/임팩트 프레임 (양면 분석 또는 단순 비교)
두 가지 모드:

**양면 분화 모드** (사건이 종목군을 K자 분기시킬 때):
"K자 분화"·"상승 vs 하락" 양면 프레임으로 1) 긍정 효과 2) 부정 효과 bullet 3~4개씩. 끝에 **종목별 4그룹 분류표** (분류/종목/논리 3컬럼) — 동반상승·알트 프리미엄·직격탄·노이즈.

**단순 비교 모드** (두 대안의 구조 차이를 보여줄 때):
소제목을 "차이점"으로 두고 1)/2) 각 옵션의 특징 bullet + `→` 한 줄 요약만. 4그룹 분류표·"핵심:" 마무리는 생략. 비교 자체가 액션 가능 정보라 추가 분류표는 군더더기.

### 5. 역사적 사례 (3개 묶음)
유사 패턴 3개를 패턴화. 단일 사례가 아닌 묶음. 두 가지 포맷:

**압축형**: 각 사례를 `- 패턴명 → 한두 줄 설명` 단일 bullet로. 분석 일관성·간결성 우선일 때.

**확장형**: `1) 2) 3)` 중제목 + `-` bullet로 분리. 각 사례를 시각적으로 구분하고 싶을 때. 공통 패턴은 `* 공통 패턴 3개` 별표 마커로 묶어 별도 블록 처리. 예:
```
1) ARKK + Tesla (2020)
  - …
2) KWEB + Alibaba (2014)
  - …
```
공통 패턴 bullet은 `→` 화살표 없이 `-` 사실 묶음으로 닫음.

### 6. 트레이딩 함의 (4개 bullet)
- bullet 1: 시간축 단계별 흐름 (예: IPO 전 6~12개월 베타 랠리)
- bullet 2: 분기점·검증 시점 (예: IPO 후 첫 실적 가이던스)
- bullet 3: 함정·리스크 (예: "테마니까 다 오른다" 단순 베타 매수 위험)
- bullet 4: **이중 조건 마무리** — "분산 포트라면 [보완재 종목] 위주, 주도주 추종이라면 [시간축 + 정리 기준]". `/blog-us`의 (A) 이중 조건 마무리 패턴 테마 글 적용.

## 공통 규약 (일일 리뷰와 동일)
- 번호 체계: 대제목 `## 1. 2. 3.`, 중제목 `### 1) 2) 3)`, bullet `-`, 분석 도입 `→`
- **헤딩 앞 빈 줄 필수** — 모든 `##`, `###` 위에 빈 줄 한 줄
- **표 포맷: 마크다운 표(`| | |`) 사용** — `.md` 파일 가독성 우선. 네이버 paste 시 raw text 노출되면 사용자가 변환.
- 티커: `RKLB +X%` 형식
- 톤: 상대적 표현, "직격탄" 같은 강한 동사는 분명한 근거(예: 멀티플 압축 산수) 있을 때만
- **숫자 일관성**: 베이스(분모) 통일. 예: NASDAQ 상장만 vs NASDAQ+NYSE 합산을 섞지 말고 한 베이스로 통일하거나 한 줄에 두 베이스 모두 명시.

## 결론부 작성 원칙
- 사용자는 short hedge 같은 레버리지 전략 사용 X → 결론부는 **롱 온리** 기준
- 시간축 분리: 베타 랠리 동참 시점 / 정리 시점
- 이중 조건: 분산 포트 vs 주도주 추종
