# 칩플레이션 동반 폭락 deep-dive — 제목 구어체·도입 무번호·역사 연대순

(애플·MS 가격 인상→'공급부족 호재'가 '수요둔화 공포'로 반전, 칩메이커·장비주 무차별 동반 폭락. 사용자와 3회 대화로 thesis 정밀화(① 동반 폭락 사실관계 → ② 호재→공포 반전 논리 → ③ 소비 메모리≠HBM '공포 오겨냥' 반론). 본문은 거의 그대로 발행, 발행 변경 3곳 소수.)

## 분류된 패턴

### 삭제
- **제목 `[테마]` 태그 발행 시 제거** (재확인). draft `# [테마] …` → 발행 본제만.

### 어조
- **테마 제목 종결 = 단정/문어형("~빠지나") → 대화체 구어 종결("~빠져요?") + 특정 종목명 제거 일반화** [신규]. draft "메모리값이 4배인데 왜 **SK하이닉스가 빠지나**" → 발행 "메모리값이 4배인데 왜 **빠져요?**". ① 종결을 친근 구어체 "~요?"로(문어형 "~나" 회피), ② 제목 주어의 특정 종목명(SK하이닉스)을 빼서 더 보편적·관계적인 질문으로. 기존 "친근 호기심 질문형" 규칙의 강화 — 종목명 콕 집기보다 독자가 "어, 왜 빠지지?" 하고 묻게 만드는 일반 의문이 후킹에 유리.

### 표현
- (의미 있는 어휘 치환 없음 — 본문 문장 거의 그대로 발행)

### 구조
- **테마글 도입부는 번호 없이 두고 본문 섹션을 1부터 매김** [신규]. draft가 도입을 무번호로 두고 본문을 `## 2.`부터 시작(`## 1.` 누락) → 발행은 도입 무번호 유지 + 본문을 1~5로 재번호(무슨일=1, 호재악재=2, 공포타당=3, 역사=4, 트레이딩=5). 테마글 도입부는 "## 1. 도입부" 번호를 붙이지 말고 무번호 리드로 두며, **실질 본문 섹션부터 1번**으로 시작. (작성 시 도입에 번호를 붙이거나, 도입을 1로 보고 본문을 2부터 시작하지 말 것 — 도입 무번호 + 본문 1~N.)
- **테마글 역사 사례 연대순 정렬 재확인** [기존 규칙 재적용]. draft 역사 사례가 연대순이 아니어서(2017~2018 → 2021 → 2000) 발행은 연대순(2000 → 2017~2018 → 2021)으로 재정렬. 기존 "테마글 역사 사례는 연대순 정렬"의 위반→교정 사례. 부수: 공통 패턴 3 bullet은 (연대순 정렬로 맨 앞이 된) 첫 사례 직후에 배치되어 "리드 사례 + 일반화 takeaway + 확인 사례 2건" 흐름으로 발행됨.

### 사실추가
- (새로 추가된 데이터·인용 없음 — 본문 거의 그대로 발행)

## 비고
- diff의 대부분은 네이버 에디터 변환 노이즈(문장 중간 줄바꿈·볼드 마커 소실·표→평문)로 학습 대상 아님.
- 이번 글은 작성 과정에서 사용자 피드백으로 thesis를 3단계 정밀화한 것이 핵심이고(K자 분화 → 무차별 동반 폭락 → '공포 오겨냥' 반론), 그 정밀화 결과가 발행본에 거의 그대로 반영됨. [[feedback_no_causal_fitting]] 원칙(결과에 맞춘 인과 짜맞추기 금지, 확인 사실 vs 시장이 든 공포 분리) 적용 사례.
