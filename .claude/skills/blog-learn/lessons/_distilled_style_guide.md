# 도담아빠 블로그 — 발행본 학습 통합 스타일 가이드

`/blog-learn`으로 누적된 lesson들에서 추출한 정수 규칙. blog-kr/blog-us/blog-thematic 작성 시 reference로 읽음.

**갱신 정책**: `/blog-learn` 실행 시 모든 lesson을 다시 읽고 처음부터 재작성. 중복·모순 통합.

**우선순위**: 본 가이드 ↔ blog-{kr,us,thematic} SKILL.md 충돌 시 본 가이드가 우선.

**글 유형 분기 (가장 중요)**:
- **일일 리뷰(blog-kr, blog-us)** = 단정형 어미 "~임/~됨/~함", 압축 호흡
- **테마 deep-dive(blog-thematic)** = 존댓말 "~입니다/~있습니다", 서술 호흡

---

## 삭제 (이런 건 빠진다)

- **시장 뉴스 표는 어닝(개별기업) 위주 2~3행으로만** [일일 리뷰]. 국제·정치·지정학 행은 표에서 빼고 매크로·원자재 본문 라인에 녹임.
- **방향이 부정 일색인 섹터 그룹은 통째 삭제** [일일 리뷰]. Sell/Short 신호만으로 채워진 약세 섹터(예: 에너지 XOM·COP·BP·TTE 전원 Sell)는 → 인사이트까지 통째 컷. 종목 수와 무관하게 *방향이 부정 일색*이면 삭제.
- **종목 2~3개짜리 후순위 섹터 그룹 만들지 말 것** [일일 리뷰]. 메인 테마에서 멀거나 종목 수 부족하면 통째 생략.
- **그룹 안 약세 outlier 단일 종목 삭제** [일일 리뷰]. 특징주 등 긍정 신호 묶음에 끼인 약세 종목(예: SYM Short Sign)은 빼고, 그 종목을 가리키던 → 인사이트의 부정 절("~ 반면, ~은 Short 지속")도 함께 삭제. 묶음은 Long·긍정 신호만 남김.
- **종합 판단 마지막 줄의 중복 트리거 제거** [일일 리뷰]. 매크로 종합·앞선 bullet에 이미 등장한 트리거(예: USD/JPY 엔캐리)는 마지막 모니터링 줄에서 중복 언급하지 말 것.
- **메인 섹터 그룹은 카테고리 엄격** [일일 리뷰]. 그룹 성격과 어정쩡한 종목(예: 반도체 AVGO를 하드웨어 그룹에)은 bullet에서 빼되, 종합 판단 산문에는 남겨도 됨 — 그룹 bullet은 엄격히, 산문은 느슨히.
- **데이터 한 줄 끝의 짧은 해석어 자제** [일일 리뷰]. "BITCOIN ... — 차익실현" 같은 단어 한두 개짜리 해석은 빼고 데이터만 남김.
- **점수 표기 금지** [공통]. 매크로 종합 라인이든 종합 판단이든 "매크로 28점" 같은 점수 언급 제거. 등급 단어만.
- **`* 공통 패턴 N개` 별표 마커 삭제** [테마]. 역사적 사례 섹션 끝의 공통 패턴 묶음은 마커 없이 자연 bullet 이어쓰기.
- **영문 제목·날짜 괄호 표기 삭제** [테마]. "Power Equipment ... (2026년 5월 29일)" → "전력기기 요즘 왜 이래요? — KR vs US 디커플링" 같이 한글 캐주얼 + 핵심 부제.

## 어조 (이렇게 다듬는다)

- **일일 리뷰 종합 판단 = 단정형 어미** — "~ 트리거.", "~ 배경이 됨.", "~ 작용하고 있음.", "~ 부분임.", "~ 필요가 있음."
- **한정 부사 "만"으로 대비 강조** [일일 리뷰]. "초장기물 소폭 상승" → "초장기물만 소폭 상승". 다른 항목과의 대비를 또렷하게.
- **이중조건 템플릿 회피** [일일 리뷰]. "포트폴리오를 다양하게 구성한다면 ~지만, 주도주 중심의 매매를 할 경우 ~" (A) 모드 그대로 X. 다단락 흐름형 자기 호흡으로.
- **테마글 = 존댓말 + 서술형 풀어쓰기** — "조정 국면입니다", "다른 양상을 보이고 있습니다", "고민을 해야할 시점에 놓였습니다". 일일 리뷰 단정형과 명확히 구분.
- **친근한 호기심 자극 제목** [테마]. "전력기기 요즘 왜 이래요?"처럼 질문형 + 캐주얼 한국어 본제 + 분석 키워드 부제.
- **제목 본제 = 본문이 실제로 푸는 가장 구체적인 인과 퍼즐** [테마]. 일반적 시장 *상태* 묘사("성장은 식는데 물가는 뜨겁다?")보다, 본문 분석의 핵심 발견을 구체적 사건·고유명사로 박은 인과형 질문("미·이란 전쟁으로 오른 유가, 물가에 영향은?")으로. 부제(지표·종목 나열)는 유지해 범위 보조. 발행 단계에서 사용자가 자주 손대는 지점이므로 draft 제목부터 클라이맥스를 끌어올릴 것.
- **독자 가치 제안 한 문장** [테마]. 도입부 마지막에 "투자자들이 이 섹터를 과연 홀드해야 하는가" 같이 독자의 현재 의사결정 상황을 명시.

## 표현 (이렇게 치환한다)

- **시장 뉴스 표 분류 어휘 = "개별기업"** [일일 리뷰]. 기업 실적 행도 분류 라벨은 "개별기업" 사용. "어닝"을 분류명으로 쓰지 말 것(시트 분류 체계와 일치).
- **분리 어휘 → 통합어** — "비철·은" → "산업 금속". 묶이는 원자재·지표는 한 단어로.
- **데이터 2~3개 묶음 → "내러티브"** — "디스인플레+장기 금리 미세 하락" → "디스인플레 내러티브". 줄줄이 나열보다 한 단어로 압축.
- **메타 동사 "정렬" 자제** — "양대 어닝이 ~로 정렬" → "~의 상향 가이던스로". 직접 명사구로.
- **압축 동사 → 서술 동사** [테마]. "손꼽혀온" → "상승 가도를 달려오던". 형용사적 압축보다 서술적 풀어쓰기.
- **분류·평가어 한 단어 끼우기** [테마]. "7~17% 고점대비 조정" → "7~17% 고점대비 정도의 안정 조정". 폭에 대한 1단어 평가("안정"·"깊은") 추가로 독자 1차 인식 보조.

## 구조 (이렇게 재배치한다)

- **5섹션 = 1.시장 뉴스 → 2.매크로 → 3.지수 → 4.섹터 → 5.종합 판단** [일일 리뷰]. 경제지표는 별도 번호 없이 1번 시장 뉴스 표에 행(분류: "경제지표")으로 통합, 표 아래 `→` 한 줄로 3축 해석. 발표 없으면 행 생략.
- **미장 일일 리뷰 제목** — "Global Market Review (YYYY년 M월 D일)" 영문 + 괄호 날짜. (테마글은 한글 캐주얼이라 정반대 — 글 유형별 별개 규칙)
- **핵심 인사이트 구절 볼드 처리** [일일 리뷰]. 각 → 인사이트와 종합 판단의 핵심 명사구("AI Capex 수혜 하드웨어로 매수가 확산", "시장 폭이 ~로 좁아진 구간" 등)를 `**볼드**`로 강조. draft 단계에서 결론 구절에 볼드 적용.
- **종합 판단 = 다단락 흐름 (이중조건 X)** [일일 리뷰]. 5단락 인과 사슬:
  1. 매크로 우호 요인 통합 (지표 4~5개를 한 단락에 묶어 "재랠리 배경" 평가)
  2. 잔존 부담 한 줄 (금리·환율 등 짧게)
  3. 어닝/이벤트 트리거 + 메인 테마 평가
  4. 한 가지 고민 (양면성)
  5. 본인 액션 + 모니터링 워치 (앞서 쓴 트리거는 중복 금지)
- **테마글 도입부 = 4~5문장 펼침** — 양상 묘사 + 폭 비교 + 독자 가치 제안 + 글 안내. 압축형 X.
- **테마글 데이터 표 끝 = 데이터 요약 + 캐주얼 1차 가설** — "단순히 생각하자면 ~ 일 수도 있음" 같이 1차 해석을 던지고 다음 섹션에서 deepen하는 다리.
- **테마글 역사적 사례 = 별표 마커 없이 공통 패턴 이어쓰기** — 1)/2)/3) 사례 다음 줄에 그대로 공통 패턴 bullet 3개.

## 사실추가 (이런 맥락이 보태진다)

- **인지도 낮은 종목 bullet에 `- 사업영역` 꼬리표** [일일 리뷰]. "DDOG ... - 서비스 모니터", "CRWD ... - 사이버 보안" 처럼 덜 알려진 종목 끝에 무슨 회사인지 한 단어 태그. 전 종목 아님 — 인지도 낮은 종목만 선별.
- **매크로 4~5지표 묶어 "재랠리/조정 배경" 평가** [일일 리뷰]. 산발 나열 X, 종합 판단에서 한 단락 묶음으로 통합 평가.
- **시장 폭·포트폴리오 안정성 trade-off 메타 인식** [일일 리뷰]. "강한 것만 오르는 현 구간에서는 포트폴리오의 안정성은 줄어들겠지만" 같이 시장 폭 좁아짐과 안정성 약화를 명시.
- **추세 맥락 + 모니터링 워치 페어** [일일 리뷰]. "AI S/W 하락 추세의 종료 또는 랠리 전환을 눈여겨 볼 필요" 처럼 추세 한 마디 + 워치 포인트를 본인 액션과 짝지음.
- **독자 행동 유발 한 문장** [테마, 도입부]. 독자가 이 글로 답하게 될 현재 의사결정 질문을 명시.
- **데이터 표 끝 1차 가설** [테마]. 본격 분석 전 가벼운 1차 해석을 한 줄로 던지고 다음 섹션에서 더 깊이 푸는 흐름.
