# 7월 넷째주 실적 시즌 프리뷰 — 제목 보편화·비중 경로 stale·별표 마커 금지

본문 구조(6섹션)·표 2개·컨센서스 수치·위스퍼 넘버 부재 고지·GAAP 함정 섹션은 **전부 그대로 발행**.
변경은 ① 제목 전면 교체 ② 비중 수치 정정(35%→25%) ③ 메모리 인용 수치 삭제 ④ 표기·지칭 다듬기 4갈래.

## 분류된 패턴

### 삭제
- **제목의 개별 종목명 + 인과 퍼즐 통삭** (② 주제 일관성) — draft "IBM을 무너뜨린 메모리 사재기, 다음주 실적이 증명해줄까요?" → 발행 "7월 넷째주 실적 시즌, 시장의 분위기를 바꿀까?". 캘린더·프리뷰형은 특정 종목의 인과 퍼즐을 제목에 걸지 않는다. 본문의 IBM 인과는 도입부가 이미 전담하므로, 제목까지 IBM에 걸면 글의 범위(다음주 전체)를 좁게 오인시킨다.
- **"8.95%" 지수 낙폭 수치 삭제** (③ 진정성) — "반도체 고점론에 8.95% 급락하는 패닉" → "반도체 고점론에 급락하는 패닉". 7/13 -8.95%와 7/16 -6.37%가 연속된 국면이라 **특정 낙폭을 집으면 어느 날을 가리키는지 모호**해진다. 연속 급락 국면에선 수치 없이 정성 서술.
- **"7월 13일" 날짜·"국장" 한정어 삭제** — "지난 7월 13일 패닉에서" → "지난 패닉에서". 본인 포지션 경로의 특정 날짜를 본문에 박지 않음.
- **`* 공통 패턴 세 가지` 별표 마커 블록 삭제** (④ 가독성) — 산문 연결로 대체. 네이버 에디터에서 `*` 마커가 렌더되지 않는 문제이자, 기존 §3.3 "역사 사례는 별표 마커 없이 이어쓰기" 규칙의 확장 적용.

### 어조
- **무매매 유지 결론의 완곡 추정 → 근거 부재 단정** — "35%까지 줄여둔 자리 **그대로 다음주를 맞이하는 게 맞다고 봅니다**" → "25%까지 줄여둔 자리에서 **변화를 꾀할 근거가 아직 없습니다**". 같은 무매매인데 "맞다고 봅니다"(내 의견)에서 "근거가 없습니다"(사실 판정)로 이동 — 추세추종 규율이 의견이 아니라 조건이라는 톤.

### 표현
- **앤스로픽 → 앤트로픽** (회사명 표기 확정, 2026-07-05 lesson과 동일).
- **캡엑스 → CapEx** — 단 §6 분기점·전략 지목에서만 영문 표기로 승격("알파벳의 **CapEx** 라인", "**CapEx 투자**를 또 올리면"). §6 안에서도 금액을 서술하는 문장("올해 캡엑스를 1,750~1,850억 달러에서")과 §1 표의 "캡엑스 상향 여부"는 한글 유지. 즉 **지표명으로 콕 집을 때만 CapEx**.
- **함의부 지칭을 은유에서 실제 현상명으로** — "함정은 **IBM을** 메모리 호재로 곧장 번역하는 것" → "**IBM의 실적을**"; "**IBM의 편지는** 지금 수요가 강하다는 증거지" → "**IBM의 실적에서의 메모리 당기기는**". §4에서 "편지"라는 환유를 썼더라도 §6 함의부에선 실제 현상명으로 되돌린다(단 "편지 한 장은 추세가 아닙니다"는 유지 — 수사가 아니라 논지 자체일 때는 남음).
- **사례 bullet 도입어 병렬 통일** — 3) 테슬라 "**그런데** 발표 당일 7.49% 하락" → "**결과는** 발표 당일 7.49% 하락". 1) IBM의 "결과는 하루 25% 하락"과 형태를 맞춤.

### 구조
- **공통 패턴 도입을 직전 사례의 꼬리에 산문으로 연결** (④ 가독성) — draft는 3) 테슬라 하위 bullet에 있던 "최근 세 번의 인도량 발표에서 세 번 다 주가가 빠졌습니다"를, 발행은 3) 밖으로 빼내 "…빠졌습니다. **공통 패턴 세 가지는 다음과 같습니다.**"로 잇고 bullet 3개를 붙였다. 별표 마커 블록이 하던 '묶음' 역할을 문장이 대신함.

## 이번 학습의 핵심 — 인덱스를 믿고 본문을 안 열었다

비중 오기의 원인은 "메모리가 낡아서"가 **아니다**. `project_cash_path.md` **본문은 이미 25%로 정확**했다(7/16 미장 줄: 30%→25%). 낡은 건 `MEMORY.md`의 **인덱스 한 줄**이었고("최신: 2026-07-13 국장 주식 35%"), draft가 그 요약만 보고 본문을 열지 않은 채 35%를 앵커했다.

→ 규칙: **비중은 MEMORY.md 인덱스가 아니라 `project_cash_path.md` 본문의 최신 세션 줄을 열어 확인한다.** 인덱스 요약은 구조적으로 갱신이 늦는다. (이번에 인덱스 줄과 본문 How-to-apply 양쪽에 경고를 박아둠.)

같은 파일에 이미 있던 경고 — "다줄 경로에서 최신 세션 줄 앵커"(2026-07-08 사례) — 가 **인덱스 vs 본문 층위에서 반복**된 셈이다.
