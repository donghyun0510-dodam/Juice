# 도담아빠 — 당일 증시 뉴스 수집·검증·기록 규약

증시 리뷰 시트의 **1.시장 뉴스 섹션(수동 입력 영역)**을 채우기 위해, 당일 뉴스를 ① 찾고 ② **날짜를 교차검증**하고 ③ 분류해 출력하고 ④ 승인 시 시트에 기록한다. 일일 리뷰 본문 작성은 `/blog-kr`·`/blog-us`.

## 🚨 절대 원칙 — 사실·날짜 검증 (위반 시 항목 폐기)
- **지어내지 말 것.** 각 뉴스는 **원기사 출처 + 발행일**을 확인한 것만.
- **날짜 교차검증 필수.** 후보 헤드라인이 *대상일자에 실제 발생/발표*된 것인지 원기사에서 확인. 과거 기사·일반 전망글이면 **제외**하거나 "○/○ 발표분"으로 명시. (사례: 한은 '반도체 성과급 물가' 보고서는 6/17 발표 → 6/18 뉴스로 넣으면 안 됨.)
- **비현실적 수치 플래그/제외.** 예: 삼성전자 분기 영업익 92조 같은 값은 fetch 오류일 수 있으니 그대로 박지 말 것.
- 불확실하면 ⚠️ 표시하고 사용자에게 확인 요청.

## 1. 입력 파싱
- 날짜: 인자 없으면 **오늘**. 시장: `KR`(국장) | `US`(미장/글로벌). 둘 다 순서 무관.
- 미장은 보통 *전 거래일(현지)* 마감분이 대상 — 날짜 해석 시 세션 기준 유의.

## 2. 뉴스 수집 (시장별 방법이 다름)
- **US (글로벌)**: `WebSearch`가 US 중심이라 잘 잡힘. `"stock market news <date>"`, `"FOMC/earnings/Fed <date>"`, 개별 종목 어닝 등으로 검색.
- **KR (국장)**: `WebSearch`는 **US-only라 한국 당일 뉴스에 약함** → **`WebFetch`로 한국 매체 페이지를 직접** 가져온다:
  - `https://markets.hankyung.com/` — 마감·수급·종목·증권사 코멘트
  - `https://www.hankyung.com/economy` — 정책·산업·거시
  - 보조: 인포스탁(`infostock.co.kr`), 연합인포맥스, 머니투데이 등
  - ⚠️ 이들 페이지는 **최근 며칠 기사를 섞어** 보여줌 → 반드시 3단계로 날짜 확인.
  - (`finance.naver.com`·`yna.co.kr`는 fetch 차단되는 경우 있음 — 막히면 한경 우선.)

## 3. 날짜 교차검증 (이 명령어의 핵심)
수집한 각 후보 뉴스마다:
- `WebSearch`(제목 키워드)나 원기사 URL `WebFetch`로 **발행일**을 확인.
- **대상일자 == 발행일** → 채택.
- **대상일자 ≠ 발행일** → 제외, 또는 시장에 계속 영향 주는 큰 건이면 "(○/○ 발표분)"으로 날짜를 명시해 구분.
- 교차검증이 끝까지 안 되는 항목은 **⚠️ 미검증**으로 표시(임의 채택 금지).

## 4. 분류 (시트 구조 + 뉴스 원칙)
- **공통 원칙**:
  - **단순 등락은 뉴스가 아니다.** 종목·섹터의 단순 주가 등락(예: "반도체 +12%", "○○ 급등/급락")은 **시트 6.섹터/종목 섹션에서 별도로 다뤄지므로 시장 뉴스에 넣지 않는다.** 시장 뉴스에는 **그 등락을 유발한 원인 뉴스**(실적·가이던스·규제·M&A·지정학 등)만 넣는다. 예: "마벨 +12%"는 빼고, "액센추어 가이던스 하향+연방발주 둔화"는 넣음.
  - **KR 국장**: **지수·수급은 제외**, **정책·개별기업·수출입동향·산업 이슈 등 시장영향** 중심. 규제·M&A·승인 뉴스는 **행위주체(정부)가 아니라 영향받는 기업 기준**으로 분류.
  - **US 글로벌**: **어닝(개별기업) 위주** + 시장영향 큰 국제·정책 이슈(FOMC·지정학 등).
- **시트 카테고리**:
  - 국장: `정책` / `개별기업`
  - 글로벌: `국제` / `정치` / `개별기업`
- 보통 **5±건**. 한 카테고리에 여러 건이면 묶는다.

## 5. 출력 (먼저 채팅에 보여주고 승인받기)
각 항목 형식: `[분류] 헤드라인 — 핵심 한 줄 (출처·날짜)`. 검증 못 한 항목은 `⚠️`.
사용자가 "몇 번 합격" 식으로 고를 수 있게 번호를 매겨 제시한다. (바로 시트에 쓰지 말고 승인 먼저.)

## 6. 시트 기록 (사용자 승인 후, 또는 `--write`)
승인된 항목만 `증시 리뷰_YYMMDD` 시트의 1.시장 뉴스에 기록.
- 워크시트: `국장` | `글로벌`. 컬럼: A 단계 / B 주제 / C 체크포인트 / D 내용 / E 비고.
- **행 매핑(템플릿 고정 — 행 삽입 금지, 기존 행 D/C에 기입)**:
  - **국장**: 2행 `정책`, 3행 `개별기업`
  - **글로벌**: 2행 `국제`, 3행 `정치`, 4행 `개별기업`
- C(체크포인트) = 토픽/종목명 짧게, D(내용) = 핵심 한 줄.
- **같은 카테고리(주제)여도 체크포인트가 다른 별개 사건이면 한 셀에 묶지 말고 행을 나눈다.** 예: 정책에 '원전'·'레버리지 ETN' 2건이면 → 2행 정책/원전, 3행 정책/레버리지 ETN, 4행 개별기업/삼성전자처럼 각자 행에 B(주제)·C(체크포인트)·D(내용)를 채움. D 셀 `•` 줄바꿈 누적은 **정말 같은 토픽의 세부 항목**일 때만(서로 다른 사건 묶기 금지).
- 한 섹션의 가용 행(국장 2~4행, 글로벌 2~4행)을 넘어 건수가 많으면 사용자에게 우선순위를 물어 추림(행 삽입은 아래 경제지표 섹션을 밀어내므로 지양).
- 인증·기록은 `gspread` + `sheet_auth.get_credentials()` (레포 루트에서 실행). 한글 인코딩은 임시 .py 파일(UTF-8)로 실행하거나 `sys.stdout.reconfigure(encoding='utf-8')`.
- 시트가 아직 없으면(daily_review 미실행) 사용자에게 알림.

## 7. 보고
기록 후, 어느 시트·행에 무엇이 들어갔는지 요약. 미검증·제외한 항목과 그 이유(날짜·수치)도 한 줄로.
